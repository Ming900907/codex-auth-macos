# Plan

## Goal
做一个 Codex 菜单栏应用，用于本地账号切换和额度查看。

## Phase
MVP 完成后的 UI 打磨。

## Scope
- 菜单栏入口
- 账号列表
- 本地账号切换
- 当前账号 `5h / 周` 剩余额度展示
- 手动刷新当前状态
- 现有菜单栏界面的视觉与文案调整
- 参考图驱动的深色毛玻璃 UI 重做

## Non-goals
- 云同步
- 团队协作
- 多设备同步
- 历史图表
- 自动切号
- 重构账号切换、登录与额度读取核心逻辑
- 改动除 `MenuBarContentView.swift` 之外的实现文件

## Approach
优先复用 Codex CLI 现有本地认证与本地账户存储，不引入额外账号体系。
本轮只在现有 SwiftUI 结构内做最小 UI 改造，优先复用已有账号历史额度快照，不增加新的核心模型层。
控制中心参考图驱动深色半透明卡片语义，避免网页式按钮和浅色面板。

## Risks
- 本地数据结构可能随 CLI 版本变化
- 切换行为可能依赖 Codex CLI 进程刷新
- 刚切换后额度数据可能暂时还是旧快照

## Acceptance
- 能查看本地账号列表
- 能切换当前账号
- 能看到当前账号 `5h / 周` 剩余额度
- 能看到英文文案、毛玻璃风格与压缩后的用量区
- 用量百分比带进度条，颜色按绿/黄/红区分
- 未激活账号可点击展开并查看已有额度信息
